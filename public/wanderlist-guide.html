Wanderlist — guide

A friendlier doorway to your Notion list of city things to do. Browse the backlog, triage it, mark things attended, and get a heads-up the day before a deadline closes.

What it is

Wanderlist reads and writes a single Notion database of things to do — events, culture, art, movies, venues, ideas. It’s a triaged backlog: keep everything you mean to get to in one place, sort by what’s expiring soonest, check items off as Attended without losing them — and see it all on a month calendar that plots each item’s planned date and expiring deadline.

Everything is stored in your Notion. The app keeps a local copy so it works offline and installs as an app; nothing lives on a server except the one-time reminder setup below.

1 · Connect Notion

  1. Create an internal integration at notion.so/my-integrations and copy its token (starts with ntn_).
  2. Open your events database in Notion → ••• → Connections → add your integration, so it can read and write.
  3. In Wanderlist, open Settings (gear icon), paste the token and the database link, and hit Test connection → Save.

The database fields

Wanderlist expects these properties (types matter):

  • Name — title
  • Description — text
  • Link — URL
  • Category — select (e.g. Event, Culture, Movie, Art, Venue, Idea) · extendable from the app
  • Place — text (holds the resolved place name) · Map — URL (the Maps pin link)
  • Tags — multi-select · freeform (free, ticketed, outdoor, with-friends…)
  • Attended — checkbox
  • Date Added — date · Date Expiring — date (the deadline to act on)
  • Planned Date — date, optional (the day you plan to go; shown on the calendar)

Note: Place is a plain text field, not Notion’s native Maps/geo field — the app writes place names through the classic Notion API, which can’t set the geo type. The app still gives you live map search (below) and stores a Maps link in Map.

2 · Place search optional

When you type in the Place field, Wanderlist can suggest real places (Google) and store the name plus a Maps link. To enable it, add a Google Places key on the server:

  1. In Google Cloud, enable Places API (New) and create an API key. Set the API restriction to just Places API (New); leave the application restriction as None — the key is used server-side (from this site’s proxy), so a Websites/IP restriction would block it.
  2. Add it to your Vercel project as an environment variable:
GOOGLE_PLACES_KEY = your-google-places-key

Without the key, the Place field simply becomes a plain text box — you type the place name yourself and everything else still works.

3 · Email reminder optional

Get one email the day before something’s Date Expiring (and not yet attended). It’s sent by a daily server job, so it reaches you even when the app is closed. Turn it on, and set the address, in Settings → Email reminder — but first do this one-time server setup:

  1. Create a Resend account and API key (the free tier is plenty; you can send from onboarding@resend.dev to your own verified address, or verify your own domain).
  2. Create a Vercel KV store on the project (Storage → KV) — it auto-adds its KV_REST_API_* env vars. This holds your reminder prefs so the cron can read them.
  3. Add these environment variables in Vercel:
WANDERLIST_NOTION_TOKEN = ntn_… (integration token, shared with the DB) WANDERLIST_DB_ID = your database id RESEND_API_KEY = re_… REMINDER_FROM = Wanderlist <onboarding@resend.dev> CRON_SECRET = any-long-random-string (optional, guards the endpoint)
  1. The daily job is already wired in vercel.json (runs each morning). Once the vars are set and you’ve toggled the reminder on in Settings with your email, you’re done.

The reminder address and name are editable in the app (saved to KV, gated by your Notion token so only you can change them). Secrets stay in Vercel env vars and never touch your browser.

4 · Everyday use

  • Add anything with just a name — flesh it out later.
  • Filter with the To-do / Attended / All segment; search by text, category, place, or tags; tap any chip to filter by it.
  • Sort by expiring-first (default), recently added, or A–Z. Expiring-soon items float up with a warm pill.
  • Hit the round ✓ to mark something attended — it stays in the list, just checked off.
  • Calendar (the calendar button): a month grid where each day shows up to two dots — a Planned dot and an Expiring dot. Tap a day to see its entries in an agenda below.
  • Themes from the sun/moon button — six palettes (Solarized, Monokai Pro…); installs as an app and works offline.
